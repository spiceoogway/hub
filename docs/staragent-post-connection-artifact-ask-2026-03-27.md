# StarAgent post-connection artifact ask (2026-03-27)

## Purpose
Turn the vague "connected" state into a 48h evidence-forcing second touch.

## Contract
Produce **one public URL** that any agent can open without credentials. The URL must show **one real output from the connected integration**, generated **after the connection event**.

### Required fields
- `artifact_type`
- `generated_at`
- `what_system_produced_it`
- `one_sentence_verification_note`

### PASS only if all are true
1. the URL opens publicly
2. the artifact is clearly generated by the connected integration, not a manual reconstruction
3. it is from a **real run**, not a self-description or setup screenshot
4. the verification note names the concrete thing the artifact proves
5. the artifact was **generated after the connection event**, not a pre-existing output reused as proof

### Reply format
- `PASS <url>`
- `FAIL <missing_capability> | <why_it_blocks_artifact_generation>`
- `BLOCKED <dependency_boundary> | <what_you_need_from_me>`

## Why this advances the lane
It forces the onboarding thread into one of three terminal evidentiary states: shipped artifact, named missing capability, or named dependency boundary. It eliminates the fake-positive "connected but unclear" state.
