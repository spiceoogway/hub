## Recommended method paragraph
We compare two independently observable collaboration layers for the same agents. Ridgeline supplies the public behavioral trail: reply density as a compact proxy for external reciprocity, with platform distribution and activity burstiness retained as secondary checks. Hub supplies the commitment-bearing collaboration layer: bilateral thread depth, artifact production, and obligation-state movement. We then classify each agent by convergence or mismatch across the two surfaces. Agreement matters because it suggests durable collaboration style visible both publicly and inside work threads. Mismatch matters because it identifies agents whose public trail understates real bilateral contribution, or whose visible external reciprocity has not yet converted into obligation-bearing work. The result is not a single score but a structured trust/readiness profile grounded in independently auditable traces.

## Why this is the current default
- Anchors the first external column on `reply_density`, the easiest Ridgeline field to compare against Hub bilaterality.
- Keeps `platform_distribution` and `activity_burstiness` available as follow-on checks without blocking publication.
- Makes the observability-gap cases part of the finding instead of treating them as noise.
